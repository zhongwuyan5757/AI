# FiveSeven AI 最小学习路径系统 v1.0

---

## 1. 路径系统的目标与原则

### 为什么学习路径比零散内容更重要

FiveSeven AI 已有 169 篇教程、500+ Prompt、3 个专栏、3 条日程式路径。内容不少，但用户面对的核心问题是：**打开网站后不知道该先看什么、看完一篇后不知道下一篇该看什么。**

零散内容像一堆散落的拼图。用户每次来都要自己决定「今天看哪块」，大部分人会在第 2-3 次访问后放弃。

学习路径的价值不是「内容更多」，而是：
- 替用户做了选择：不用想，跟着走就行
- 制造了完成感：每走完一步都知道自己进步了
- 建立了回访理由：「我还有 3 步没完成」比「这网站内容不错」的黏性强 10 倍

### 为什么当前阶段只适合做最小路径系统

现有系统的三个现实约束决定了必须从最小方案做起：

1. **内容分布不均匀**：169 篇教程中 70 篇在三大专栏里，99 篇是散落的。不是所有岗位都有足够内容支撑一条完整路径——产品运营有丰富的教程和 Prompt，但数值策划和新媒体的教程几乎是空白
2. **路径基础设施已有雏形**：app.js 中已有 `LEARNING_PATHS` 对象和 `renderLearningPath` 渲染函数，columns.json 有阶段化结构——不需要从零开发，只需要在现有框架上扩展
3. **没有用户行为数据**：在没有用户完成率、跳出率数据之前，设计 20 条路径是盲目的。先做 2 条，验证模式，再扩展

### 什么是「成长感」，为什么它对 FiveSeven AI 很重要

成长感 = 用户在每次访问后都能感知到自己比上次更强了。

它不需要积分、徽章、排行榜这些复杂系统。成长感的核心机制只有三个：

| 机制 | 实现方式 | 成本 |
|------|---------|------|
| **位置感** | 让用户随时知道「我在第几步，一共几步」 | 文案 + 结构 |
| **产出感** | 每一步结束时用户产出了一个具体东西 | 内容设计 |
| **方向感** | 完成当前步骤后立刻看到下一步是什么 | 导航引导 |

FiveSeven AI 当前缺的不是内容量，而是成长感。用户读完一篇教程后看到的是「相关推荐」——这是信息流逻辑，不是学习逻辑。学习逻辑应该是：「你刚完成第 2 步，下一步是 XX，你将学会 YY。」

## 2. 推荐优先做的 2 条路径

### 选择逻辑

从 9 个岗位中选路径，必须同时满足三个条件：
1. **已有内容能覆盖 5 步中的至少 3 步**（不需要全部新写）
2. **目标人群是网站最大用户群**（最大化路径的服务人数）
3. **路径产出物具体且可验证**（用户能感受到自己做出了东西）

### 路径 A：游戏运营 AI 上手路径

| 维度 | 说明 |
|------|------|
| **路径名称** | 游戏运营 AI 上手路径 — 5 步搞定日常 80% 的 AI 提效场景 |
| **适合人群** | 产品运营（product-ops）：活动策划、数据分析、文案撰写、SOP 整理 |
| **为什么优先** | 运营是游戏公司人数最多的岗位，AI 使用频率最高（文案、数据、方案占日常 60%+）。现有内容中运营相关的教程最多（t001、t002、t004、t008、t009、t016），Prompt 也最多（45+ 个运营场景 Prompt），能覆盖完整的 5 步路径 |
| **与现有栏目联动** | 每步关联：实战指南（核心教学）+ Prompt 库（即用模板）+ 工具库（工具选型）。路径终点接入 Claude 专栏的进阶内容 |

### 路径 B：AI 内容生产全流程

| 维度 | 说明 |
|------|------|
| **路径名称** | AI 内容生产全流程 — 从选题到分发的完整内容工作流 |
| **适合人群** | 新媒体 + 品牌经理 + 投放经理 — 所有需要「大量产出内容」的岗位 |
| **为什么优先** | 内容生产是跨岗位最高频的 AI 使用场景。新媒体、品牌、投放三个岗位加起来覆盖了团队中「最需要 AI 帮忙写东西」的人群。现有教程中 t005、t008、t010、t012 直接覆盖内容生产场景，Prompt 库中内容创作类模板超过 80 个 |
| **与现有栏目联动** | 每步关联：实战指南（流程教学）+ Prompt 库（文案模板）+ 工具库（生产工具）。路径终点接入 Nano Banana 专栏（视觉内容）和 OpenClaw 专栏（内容自动化） |

## 6. 轻量成长感机制

不做积分系统、不做注册登录、不做数据库。只用以下 4 种零成本方式制造成长感。

### 6.1 文案制造成长感

**每步开头的定位文案**：

| 步骤 | 定位文案 |
|------|---------|
| 第 1 步 | 「这是你的 AI 之旅起点。完成后你将能让 AI 帮你写出第一份可用的工作文档。」 |
| 第 2 步 | 「你已经会和 AI 对话了。现在升级——让 AI 帮你产出完整的策划方案。」 |
| 第 3 步 | 「方案你会做了，但运营离不开数据。这一步让 AI 成为你的数据分析搭档。」 |
| 第 4 步 | 「单篇可以了，但运营的日常是重复性内容。学会批量生产，效率翻倍。」 |
| 第 5 步 | 「你已经掌握了 4 种核心 AI 技能。最后一步，把它们串联成你自己的工作流。」 |

**关键文案规则**：
- 每步开头第一句回顾上一步的成果（「你已经 XX 了」）
- 第二句指出下一个能力缺口（「但你还缺 YY」）
- 第三句承诺本步的产出（「完成后你将拥有 ZZ」）

### 6.2 步骤顺序制造成长感

**递进设计原则**：

```
能力范围：     点        →      面        →      线        →      量        →      流
具体表现：  写一段话     → 写一份完整方案  → 分析一份数据   → 批量产出内容   → 串联成工作流
复杂度：    单次对话     →   多轮迭代     →  数据+文本     →  模板化复用    →  跨工具串联
独立性：    跟着教程做   →  半独立操作    →  独立操作      → 建立自己的库   → 设计自己的流程
```

每一步的变化维度：
- **输出物从小到大**：一段话 → 一份方案 → 一份报告 → 一个模板库 → 一套工作流
- **自主性从低到高**：照着做 → 半自己来 → 自己来 → 自己设计
- **工具从单一到组合**：一个工具 → 一个工具深入 → 两个工具配合 → 多工具 + Prompt 库 → 工具链

### 6.3 相关推荐制造成长感

**不用「相关推荐」这个词。改用引导性语言**：

| 常见做法 | FiveSeven AI 的做法 |
|----------|-------------------|
| 「相关教程」 | 「完成这一步后，这些内容能帮你走得更远」 |
| 「你可能感兴趣」 | 「运营岗的同学还在用这些 Prompt」 |
| 「推荐阅读」 | 「这 2 篇是下一步的预备知识」 |
| 「热门内容」 | 「和你同阶段的用户最常用这 3 个模板」 |

**推荐位置规则**：
- 每步教程结束后，只推荐 2-3 个内容，不超过 3 个
- 推荐内容必须属于以下三类之一：本步的补充练习、下一步的预备知识、同场景的 Prompt 模板
- 不推荐与当前路径无关的内容

### 6.4 完成提示制造成长感

**使用 localStorage 记录进度**（现有系统已有此机制）。

完成一步后的提示设计：

```
┌──────────────────────────────────────────────┐
│                                              │
│  ✅ 第 3 步完成！                              │
│                                              │
│  你已经掌握了：                                │
│  · 让 AI 听懂需求                             │
│  · 生成完整策划方案                            │
│  · 用 AI 分析运营数据                ← 刚学会   │
│                                              │
│  你的进度：███████████░░░░░  3/5 步            │
│                                              │
│  下一步：批量生产运营内容                       │
│  预计 40 分钟 · 你将建立自己的 Prompt 模板库     │
│                                              │
│  [ 继续学习 →]     [ 稍后再来 ]                │
│                                              │
└──────────────────────────────────────────────┘
```

**关键设计**：
- 回顾已学会的所有能力（强化积累感）
- 标注「刚学会」的那一条（强化本次收获）
- 进度条用实心方块（比百分比更直观）
- 下一步包含具体产出物预告（制造期待）
- 「稍后再来」不是放弃，是留有余地——用户下次打开路径页会看到进度

## 8. 执行顺序建议（分阶段）

### 第一阶段：路径框架上线（1-2 天）

**目标**：让 2 条路径在网站上可见、可点击、可跟踪进度。

| 序号 | 任务 | 具体内容 | 工作量 |
|------|------|---------|--------|
| 1 | 扩展 LEARNING_PATHS 数据 | 在 app.js 中新增 2 条路径数据（含 relatedPrompts、relatedTools、outcome 字段） | 约 100 行 |
| 2 | 扩展 renderLearningPath 函数 | 在步骤卡片中渲染推荐 Prompt 和工具链接、产出物文案 | 约 40 行 |
| 3 | 步骤完成提示组件 | 完成步骤后展示回顾 + 进度条 + 下一步预告 | 约 60 行 |
| 4 | 用现有教程填充 | 先用 t001、t009、t016、t008、t015 等现有教程填充路径 | 纯数据 |

此阶段结束后：用户可以在路径页看到 2 条新路径，点进去能跟着现有教程一步步学。

### 第二阶段：核心内容补齐（3-5 天）

**目标**：补齐路径中缺失的核心教程，确保每步都有对应的高质量内容。

| 序号 | 任务 | 优先级 |
|------|------|--------|
| 1 | 新写 A1「运营日常 AI 第一课」 | 最高 |
| 2 | 新写 B1「AI 选题策划实战」 | 最高 |
| 3 | 改写 A2 t016（加入运营数据案例） | 高 |
| 4 | 新写 B2「一篇内容发 5 个平台」 | 高 |
| 5 | 新写 B3「非设计师的 AI 出图指南」 | 高 |
| 6 | 新写 A3「5 个运营必备 Prompt」 | 高 |

### 第三阶段：成长感优化（1-2 天）

**目标**：加入文案引导和完成提示，让路径不只是内容列表，而是有成长感的学习体验。

| 序号 | 任务 |
|------|------|
| 1 | 为每步添加开篇定位文案（回顾 + 缺口 + 承诺） |
| 2 | 为每步添加结束引导文案（产出确认 + 下一步预告） |
| 3 | 完成提示弹窗组件（回顾能力列表 + 进度条） |
| 4 | 路径完成后的结业引导（推荐专栏 + 推荐进阶路径） |

### 第四阶段：数据验证与迭代（持续）

**目标**：根据用户行为数据调整路径设计。

| 关注指标 | 判断标准 | 调整策略 |
|----------|---------|---------|
| 每步完成率 | 某步完成率低于 40% | 检查该步教程质量或难度跳跃 |
| 路径完成率 | 整条路径完成率低于 15% | 考虑缩短路径或降低后半段难度 |
| 步骤间跳出 | 用户在第 X 步后大量离开 | 在第 X 步和第 X+1 步之间加入过渡内容 |
| 路径页访问 → 第 1 步点击转化 | 低于 30% | 优化路径页顶部文案和价值承诺 |

此阶段依赖在路径页和教程页添加简单的 localStorage 事件追踪（无需后端）。

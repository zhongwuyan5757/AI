# FiveSeven AI 最小闭环落地方案

---

## 1. 本轮 Sprint 核心目标

**一句话目标**：7 天内，让 FiveSeven AI 从"内容展示站"变成"帮用户判断和行动的知识站"。

**具体定义"明显更强"**：

| 维度 | 当前状态 | 7 天后状态 |
|------|---------|----------|
| 工具页 | 5 行简介，跟官网首页没区别 | 3 个核心工具有完整的判断、对比、局限性说明 |
| 模型页 | 一段描述 + 标签 | 2 个核心模型有场景推荐、模型对比、性价比建议 |
| Prompt 页 | 复制就走，用不用得好看运气 | 5 个头部 Prompt 有变量填写指南、失败排查、新手/进阶版 |
| 学习入口 | 用户不知道从哪开始 | 1 条运营路径明确引导"5 步从入门到工作流" |
| 专题感知 | 没有 | 1 个专题入口页展示系统化知识结构 |

**验收标准**：任何一个运营岗用户打开网站，10 秒内能找到学习入口，30 秒内能在工具页做出选型判断，60 秒内能复制一个带使用说明的 Prompt。

---

## 2. 为什么现在不能再继续写策略

5 个 MVP 文档一共产出了：

- 3 套内容模板（实战指南 / 工具页 / Prompt 页）
- 2 条学习路径设计
- 15 个工具页 + 5 个模型页的升级方案
- 20 个 Prompt 的扩展字段设计
- 2 个专题的 24 模块知识架构
- 100+ 个评分标准、写作规则、联动逻辑

**问题**：这些全是纸上方案。网站上一个字都没改。

用户感知到的仍然是：

- 工具页 = 一段 desc + "访问官网"按钮
- Prompt = 复制走人
- 不知道从哪开始学
- 教程之间没有串联

**结论**：策略的边际收益已经为零。此刻唯一有价值的工作是把策略变成代码和数据。每多写一份策略文档，都是在拖延上线。

## 3. 最优实施顺序

### 不按文档编号顺序执行。按"用户感知提升 × 实现成本"排序。

```
最优顺序                     为什么

第 1 步：工具/模型决策化升级   ← 当前最弱的页面，视觉提升最大
         (MVP3 部分)            3 个工具 + 2 个模型，纯数据填写 + 1 个展开面板组件

第 2 步：Prompt 任务化升级     ← 用户每天都在复制 Prompt，加使用说明立刻有感
         (MVP4 部分)            5 个 Prompt，复用第 1 步的展开面板模式

第 3 步：学习路径上线          ← 给用户一个"从这里开始"的入口
         (MVP2 部分)            扩展 LEARNING_PATHS 数据 + renderLearningPath 小幅改造

第 4 步：专题入口页            ← 展示知识体系感，但不需要填满所有模块
         (MVP5 部分)            topics.json + renderTopicPage 基础版

第 5 步：内容标准作为审核参考   ← MVP1 不产出页面，作为内部参考贯穿全程
         (MVP1)
```

### 为什么这个顺序是最优的

| 顺序 | 核心理由 |
|:---:|---------|
| 工具/模型先做 | 工具页是当前最弱的页面（只有 5 行信息），升级后视觉对比最大。而且"展开面板"组件做完后 Prompt 和专题都能复用 |
| Prompt 第二做 | Prompt 是用户最高频使用的功能（525 个模板），5 个样板升级后立刻提升"可用性"感知 |
| 学习路径第三做 | 路径需要工具页和 Prompt 页已升级，这样路径中推荐的内容点进去质量才过关 |
| 专题最后做 | 专题是"聚合层"，需要底层内容（工具/Prompt/教程）先升级，专题才有东西可串联 |
| 内容标准贯穿 | MVP1 的模板和审核清单不是独立任务，而是每一步产出内容时的质量门禁 |

### 每一步如何为下一步铺路

```
第 1 步：建立"卡片展开面板"前端模式
    ↓ 复用这个组件
第 2 步：Prompt 卡片也用展开面板
    ↓ 工具+Prompt 质量就绪
第 3 步：学习路径可以放心推荐这些内容
    ↓ 路径中的教程→工具→Prompt 链路通了
第 4 步：专题聚合已升级的工具+Prompt+路径，一页展示
```

## 6. 首批样板页优先级

### 6.1 实战指南样板：本轮不改写教程

**决定**：本轮不改写任何现有教程。

**理由**：
- 教程改写是最耗时的内容工作（每篇 4-8 小时）
- 本轮的 7 天应该 100% 投入前端组件 + 数据填写
- 现有教程（t001、t009、t016）虽然不完美，但作为路径和专题中的链接内容够用
- 教程改写放在第 2 个 Sprint，用 MVP1 的模板和评分体系指导改写

**替代做法**：通过工具页、Prompt 页、路径页的升级，让教程周围的"引导层"变强。用户读教程的体验不变，但到达教程的路径和离开教程后的引导变好了。

### 6.2 工具页样板：ChatGPT

**为什么 ChatGPT 第一个做**：
- 流量最大的工具入口（90% 用户第一个接触的 AI 工具）
- 已有对应教程 t001（可以链接）
- 最常被问的两个问题——"免费版够不够""跟 Claude 怎么选"——正好由决策化字段回答
- 扩展字段在 MVP3 文档中已写好完整示例，直接填入即可

**解决的用户问题**：
- 免费版 GPT-5 够不够用？（pricingAdvice 回答）
- ChatGPT 和 Claude 怎么选？（vsAlternatives 回答）
- 它做不好什么？（limitations 回答）

**为什么适合做标准样板**：ChatGPT 的竞品对比（vs Claude、vs Gemini）覆盖了最常见的选型场景。做好这一个，Claude 和 Perplexity 的格式直接照搬。

### 6.3 模型页样板：Claude Sonnet 4.6 vs Opus 4.6

**为什么这对模型第一个做**：
- FiveSeven AI 有 19 篇 Claude 专栏教程，Claude 是站内内容最深的工具
- 用户最大的困惑是"Sonnet 和 Opus 到底怎么选"
- 这两个模型共用一个工具（Claude），对比最有实际价值
- MVP3 文档已提供了完整的扩展字段示例

**解决的用户问题**：
- 什么任务用 Sonnet、什么任务切 Opus？（vsModels 回答）
- 80% 任务用 Sonnet 就够了？（costPerformance 回答）
- 策划岗具体用哪个？（recommendedFor 回答）

**为什么适合做标准样板**：模型对比是用户最刚需的决策场景。Sonnet vs Opus 的对比做好了，后续 GPT-5 vs GPT-5.4、Flash vs Pro 直接复用格式。

### 6.4 Prompt 样板（5 个）

按优先级排列：

| # | Prompt | ID | 为什么优先 | 解决什么问题 | 用户直接价值 |
|:---:|--------|:---:|----------|-------------|------------|
| 1 | 活动策划方案生成器 | p001 | 运营岗使用频率最高，MVP4 已写好完整扩展示例 | "我想做活动但不知道 Prompt 变量怎么填" | 变量填写有 good/bad 对比，跑出来不对有排查指南 |
| 2 | PRD 需求文档生成器 | p004 | 策划岗核心交付物，67 个需求文档 Prompt 的入口 | "PRD 结构不对，输出太散" | inputGuide 教用户怎么描述需求背景，commonFailures 解决"输出太笼统" |
| 3 | 数据周报自动生成器 | p009 | 每周必做任务，提效最直观（2h→20min）| "数据丢给 AI 但输出的周报不能用" | 教用户怎么准备数据输入格式 |
| 4 | 广告文案批量生成器 | p002 | 投放岗每天都在用，批量生成是最高频场景 | "5 条文案太相似，缺乏差异" | optimizationTips 教用户追问差异化 |
| 5 | 短视频脚本生成器 | p005 | 短视频是内容产出最高频形态 | "脚本太长/太短，不符合平台要求" | 新手版 3 个变量速跑，进阶版加平台约束 |

**为什么这 5 个一起做**：覆盖了"写方案 + 做分析 + 产内容"三大任务分类，涉及 4 个岗位（运营、策划、投放、新媒体）。5 个样板做完，后续 15 个 Prompt 升级直接照搬格式。

### 6.5 学习路径样板：游戏运营 AI 上手路径

**为什么运营路径第一个做**：
- 运营岗覆盖的教程（31 篇）和 Prompt（208 条）最多，路径内容充足
- 路径的 5 步（对话→方案→数据→批量→工作流）正好串联了本轮升级的 Prompt（p001 在第 2 步、p009 在第 3 步、p002 在第 4 步）
- 路径推荐的工具（ChatGPT、Claude）正好是本轮升级的工具页

**解决的用户问题**：
- "我是运营，打开网站不知道该先看什么"
- "看完一篇教程不知道下一步该学什么"

**为什么适合做标准样板**：运营路径的数据结构和渲染逻辑验证后，内容生产路径（路径 B）只需填数据，代码零改动。

### 6.6 专题入口样板：游戏运营 AI 方案全书

**为什么运营专题第一个做**：
- 与学习路径（6.5）共享同一批目标用户（运营/策划）
- 本轮升级的 3 个工具页和 5 个 Prompt 全部在这个专题的知识架构中
- 用户从路径完成后自然进入专题深度学习，形成闭环

**解决的用户问题**：
- "这个网站有很多内容，但看不出系统性"
- "我想系统学习运营 AI，不知道有多少内容可学"

**为什么适合做标准样板**：运营专题的 4 层 12 模块中，6 个模块可直接串联已有教程和 Prompt，不需要新建内容即可上线入口页。第二个专题（AI 视觉内容生产指南）只需填 topics.json 数据。

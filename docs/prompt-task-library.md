# FiveSeven AI Prompts 栏目 — 任务解决库升级方案

---

## 1. Prompts 栏目的正确定位

### 为什么普通 Prompt 库很容易失去价值

FiveSeven AI 已有 525 个 Prompt，覆盖 22 个场景。数量不少。但从当前数据结构看，每个 Prompt 只有 6 个字段：

```
name → 知道它叫什么
scenario → 知道它属于哪个场景
recommendedModel → 知道用什么模型跑
prompt → 复制粘贴
example → 一行示例
notes → 一句提示
```

这套结构的核心问题：**用户能复制，但不知道怎么改、改哪里、改完预期什么结果、跑出来不对怎么办。**

普通 Prompt 库失去价值的 4 个典型路径：

| 阶段 | 发生了什么 | 结果 |
|------|-----------|------|
| 第 1 次 | 用户复制 Prompt，替换变量，跑了一次 | 有效果，觉得有用 |
| 第 2 次 | 用户想调整输出风格，但不知道改哪里 | 改了一个词，输出变差了 |
| 第 3 次 | 用户换了一个模型跑，效果完全不同 | 以为 Prompt 不好用 |
| 第 4 次 | 用户不再来了 | 525 个 Prompt 变成 0 个回访 |

根本原因：**Prompt 本身不是价值，用 Prompt 完成任务才是价值。** 没有使用说明的 Prompt，就像没有说明书的工具——用户拿到了，但用不好就放弃了。

### 什么样的 Prompt 系统才真正有用

一个真正有用的 Prompt 系统必须回答用户的 5 个连续问题：

1. **我要完成什么任务？** → 按任务入口而非按场景标签入口
2. **这个 Prompt 怎么填？** → 每个变量有填写指南和好/差示例
3. **跑出来应该长什么样？** → 有预期输出的参考
4. **效果不好怎么调？** → 有常见失败原因和优化方向
5. **我水平提高了，有没有进阶版？** → 有新手版/进阶版/Agent 版梯度

满足这 5 个条件的 Prompt 不叫「提示词」，叫「任务模板」。

### FiveSeven AI 的 Prompts 栏目应该解决什么问题

定位从「Prompt 展示区」变成「任务解决库」：

```
旧定位：这里有 525 个 Prompt，按场景/岗位筛选，复制就能用
新定位：你要完成什么任务？找到它，复制模板，按说明填入，AI 帮你做完
```

核心变化：

| 维度 | 旧 | 新 |
|------|-----|-----|
| 入口逻辑 | 按场景浏览 | 按任务搜索 |
| 用户心态 | 「有什么 Prompt 可以看看」 | 「我要做 XX，帮我找模板」 |
| 内容深度 | prompt + 一行 example + 一行 notes | prompt + 填写指南 + 预期输出 + 失败排查 + 进阶版 |
| 使用体验 | 复制 → 走人 | 复制 → 填入 → 验证 → 优化 → 掌握 |

## 2. 4 大任务分类

不按模型分、不按场景标签分。按用户进入时的任务意图分。

### 分类 1：写方案 — 从零到一产出完整文档

| 字段 | 内容 |
|------|------|
| **分类名称** | 写方案 |
| **适合用户** | 产品运营、产品策划、品牌经理、发行经理 |
| **高频任务** | 活动策划方案、PRD 文档、竞品分析报告、传播方案、商业计划书、KOL 合作方案 |
| **为什么优先级高** | 「写方案」是游戏团队中覆盖面最广、频率最高的 AI 使用场景。现有 525 个 Prompt 中，活动策划(34) + 需求文档(67) + 品牌营销(34) + 竞品分析(20) + 发行策划(24) + 调研分析(20) = 199 个，占比 38%。这些 Prompt 的共同特征是：输入一组条件，输出一份结构化文档。 |

**现有 Prompt 覆盖情况**：已有 199 个，内容丰富但缺少使用指南和预期输出。优先做的是给头部 Prompt 加深度，不是加数量。

### 分类 2：做分析 — 把数据和信息变成结论

| 字段 | 内容 |
|------|------|
| **分类名称** | 做分析 |
| **适合用户** | 产品运营、数值策划、投放经理、产品策划 |
| **高频任务** | 数据周报生成、用户反馈分析、投放数据复盘、数值表校验、市场趋势分析、留存漏斗诊断 |
| **为什么优先级高** | 数据分析是最能体现 AI 效率提升的场景——原来 2 小时的周报，用 Prompt 20 分钟完成。但也是最容易「跑出来不对」的场景。现有 Prompt 中数据分析(27) + 数值校验(36) + 用户分析(23) = 86 个，占比 16%。这些 Prompt 对输入格式要求高，必须有填写指南。 |

**现有 Prompt 覆盖情况**：已有 86 个。最缺的不是数量，而是「输入数据怎么准备」和「输出结果怎么验证」的说明。

### 分类 3：产内容 — 批量产出文案和创意素材

| 字段 | 内容 |
|------|------|
| **分类名称** | 产内容 |
| **适合用户** | 新媒体、品牌经理、投放经理、视频设计师 |
| **高频任务** | 广告文案批量生成、短视频脚本、社媒选题策划、公告文案、口播稿、多平台改写 |
| **为什么优先级高** | 内容生产是最高频的日常任务——新媒体岗位每天都在用。现有 Prompt 中广告文案(17) + 内容创作(34) + 运营文案(16) + 视频脚本(11) + 视频制作(16) = 94 个，占比 18%。这类 Prompt 的特点是需要批量变体和多平台适配。 |

**现有 Prompt 覆盖情况**：已有 94 个。最缺的是「不同平台的调性差异说明」和「批量生成时如何保持差异性」的指导。

### 分类 4：提效率 — 加速重复性日常工作

| 字段 | 内容 |
|------|------|
| **分类名称** | 提效率 |
| **适合用户** | 全岗位 |
| **高频任务** | 会议纪要整理、SOP 文档生成、邮件撰写、翻译润色、信息摘要、PPT 大纲生成 |
| **为什么优先级高** | 这类任务不属于任何一个岗位的「核心工作」，但每个人每天都在做。AI 提效最明显、学习门槛最低、见效最快。现有 Prompt 中 SOP 生成(20) + 效率工具(27) + 汇报展示(12) + 本地化(15) = 74 个，占比 14%。 |

**现有 Prompt 覆盖情况**：已有 74 个。这类 Prompt 最适合做「新手版 → 进阶版」的梯度设计。

### 各模块写作要求

| 模块 | 怎么写 | 绝对不能 |
|------|--------|---------|
| **taskDesc** | 一句话说清「输入什么 → 输出什么」，不超过 40 字 | 写成功能描述（「一个活动策划 Prompt」） |
| **whenToUse** | 用「当你需要 XX 时」的句式，描述具体触发场景 | 写「适合各种场景」 |
| **whenNotToUse** | 写这个 Prompt 真的搞不定的情况，推荐替代方案 | 留空或写「无」 |
| **inputGuide** | 每个变量给 good/bad 示例对比，tip 写为什么要这样填 | 只写「请填入相关信息」 |
| **exampleFull** | 必须是实际在推荐模型上跑过的，标注模型名和版本 | 编造 AI 输出结果 |
| **commonFailures** | 按「症状 → 原因 → 解决」三段式，每个不超过 2 行 | 写成通用建议（「多试几次」） |
| **optimizationTips** | 3 条以内，每条是一个可直接复制的追问句式 | 写理论（「好的 Prompt 应该...」） |
| **versions.lite** | 变量减到 3 个以内，新手 5 分钟内跑完 | 砍掉核心变量导致输出质量不可用 |
| **versions.advanced** | 在标准版基础上追加 2-3 个控制项，用 promptDiff 说明差异 | 重写一个完全不同的 Prompt |

## 5. 站内联动逻辑

### 联动原则

Prompt 不是终点站。用户在 Prompt 页复制完模板后，必须看到两个方向：
- **往前看**：「想深入学习？去看教程」
- **往旁边看**：「做完这个任务，下一个相关任务是什么？」

### 5 条联动规则

#### 规则 1：Prompt → 工具页

```
触发：Prompt 的 recommendedModel 字段
渲染：在 Prompt 页面顶部和底部展示推荐工具卡片（迷你版）
文案：「这个 Prompt 用 Claude 效果最好 → 查看 Claude 工具页」
价值：让用户知道为什么推荐这个工具，而不只是一个品牌标签
```

#### 规则 2：Prompt → 实战指南

```
触发：relatedTutorials 字段
渲染：底部「推荐搭配阅读」区域
文案：「想系统学会活动策划 AI 用法？→ AI 活动策划实战 (t009)」
价值：Prompt 解决「复制就能用」，教程解决「理解原理、举一反三」
```

#### 规则 3：Prompt → 学习路径

```
触发：relatedPaths 字段
渲染：底部路径入口卡片
文案：「这个 Prompt 是运营 AI 上手路径的第 2 步 → 查看完整路径」
价值：让散点用户进入结构化学习
```

#### 规则 4：Prompt → 相关 Prompt

```
触发：relatedPrompts 字段
渲染：底部「做完这个，试试这些」区域
文案：「活动方案做完了？接下来用这些 Prompt 做数据追踪和复盘」
逻辑：推荐的 Prompt 必须是任务链上的下一步，不是同类罗列
       示例：活动策划(p001) → 数据周报(p009) → 活动复盘分析
```

#### 规则 5：实战指南 / 学习路径 → Prompt

```
触发：教程内容中引用的 Prompt、路径步骤中的 relatedPrompts
渲染：教程步骤旁边展示「本步推荐 Prompt」迷你卡片（含一键复制）
文案：「读完这一步，直接用这个 Prompt 练习」
价值：让教程不只是阅读，而是阅读 + 操作
```

## 6. 最小上线策略

### 阶段一：给 20 个头部 Prompt 加深度（3-5 天）

**目标**：不加新 Prompt，只给第 3 节列出的 20 个 Prompt 补充扩展字段。

**工作内容**：

| 任务 | 工作量 | 质量标准 |
|------|--------|---------|
| 填写 inputGuide（变量填写指南） | 15-20 分钟/个 | 每个变量有 good/bad 对比 |
| 填写 commonFailures（常见失败） | 10 分钟/个 | 至少 2 条，按症状→原因→解决 |
| 填写 exampleFull（完整示例） | 20-30 分钟/个 | 必须在推荐模型上实测 |
| 填写 versions.lite（新手版） | 10 分钟/个 | 变量减到 3 个以内 |
| 填写其他扩展字段 | 10 分钟/个 | 按模板要求 |
| **单个 Prompt 总计** | **约 1-1.5 小时** | |
| **20 个 Prompt 总计** | **约 3-4 天** | |

**质量门禁**：每个升级后的 Prompt 发布前必须通过以下检查：
- [ ] exampleFull 在推荐模型上实际跑过
- [ ] inputGuide 的 good/bad 示例有明确差异
- [ ] commonFailures 至少 2 条且不重复
- [ ] versions.lite 的变量不超过 3 个
- [ ] whenNotToUse 不为空

### 阶段二：前端增加任务分类筛选（1 天）

**目标**：在现有筛选栏中增加「按任务类型」的一级筛选。

**改动**：

```javascript
// 在现有 renderPrompts 函数中增加一级筛选
const TASK_CATEGORIES = {
  'all': '全部任务',
  '写方案': '写方案',
  '做分析': '做分析',
  '产内容': '产内容',
  '提效率': '提效率'
};

// 筛选逻辑
if (taskFilter !== 'all') items = items.filter(p => p.taskCategory === taskFilter);
```

**注意**：525 个 Prompt 中只有 20 个有 taskCategory 字段。未标注的 Prompt 在「全部任务」中正常显示，在特定任务分类下不出现。这样可以渐进式地给更多 Prompt 打标签。

### 阶段三：升级 Prompt 卡片渲染（1-2 天）

**目标**：有扩展字段的 Prompt 展示更丰富的卡片，没有的保持原样。

**改动**：

```javascript
function renderPromptCard(p) {
  // 如果有扩展字段，渲染升级版卡片
  const hasExtended = !!p.taskDesc;

  if (hasExtended) {
    return renderExtendedPromptCard(p);  // 新函数，含展开面板
  }
  return renderBasicPromptCard(p);       // 现有逻辑不变
}
```

升级版卡片与普通卡片的区别：
- 卡片顶部增加 taskDesc 一句话
- 增加 whenToUse 标签
- 点击卡片可展开详情面板（变量指南 + 完整示例 + 失败排查）
- 底部增加版本切换（新手版/标准版/进阶版）

### 阶段四：批量扩展（持续）

**目标**：逐步给剩余 505 个 Prompt 补充扩展字段。

**优先级**：
1. 先补 taskCategory（只需 1 个字段，5 秒/个）→ 让所有 Prompt 能按任务分类筛选
2. 再补 whenToUse + whenNotToUse（2 个字段，2 分钟/个）→ 最小成本提升决策价值
3. 最后补 inputGuide + commonFailures + versions（完整扩展，1 小时/个）→ 只给高使用率的 Prompt 做

**不做的事**：不给所有 525 个 Prompt 做完整扩展。80% 的使用量集中在 20% 的 Prompt 上。头部 50 个 Prompt 做完整扩展，其余只打分类标签即可。

### 如何避免「数量很多但价值不高」

4 条铁律：

1. **不为凑数而加 Prompt**：525 个已经够多了。接下来只在现有 Prompt 无法覆盖的任务场景下才新增。新增前先检查：现有的 Prompt 改一改能不能用？
2. **新增 Prompt 必须走完整模板**：所有新增的 Prompt 必须包含 inputGuide + exampleFull + commonFailures，不接受只有 prompt + example + notes 的简版。
3. **每月淘汰低使用率 Prompt**：添加简单的 localStorage 点击计数，3 个月内零点击的 Prompt 考虑下架或合并。
4. **Prompt 质量 > Prompt 数量**：宁可 50 个高质量任务模板，不要 500 个只能复制但不会用的提示词。

### 选定专题

#### 专题 A：游戏运营 AI 方案全书

**定位**：游戏行业运营人的 AI 工作手册——从活动策划到数据复盘，每个环节给出"可直接用"的 AI 方案

**选择理由**：
- **内容密度最高**：可串联 31 篇教程 + 208 条 Prompt + 9 个工具，覆盖 product-ops 和 product-planner 两个最强岗位
- **壁垒最强**：竞品网站有"AI 教程合集"，但没有人做过"游戏运营全流程 × AI 方案"的系统化梳理
- **复用率高**：现有内容覆盖活动策划(34 prompts)、需求文档(67 prompts)、数据分析(27 prompts)、用户分析(23 prompts)
- **差异化明显**：不是"教你用 AI 工具"，而是"教你用 AI 做运营"——以任务为锚点，工具为手段

#### 专题 B：AI 视觉内容生产指南

**定位**：游戏行业视觉工作者的 AI 产出手册——从海报出图到视频制作，建立完整的 AI 辅助视觉生产线

**选择理由**：
- **工具覆盖最全**：可串联 18 个视觉类工具(image 8 + video 10)、26 篇教程(graphic-designer 15 + video-designer 11)、73 条 Prompt
- **时效性壁垒**：视觉 AI 工具迭代极快(Midjourney V7、Sora 2、Kling 3.0)，持续更新本身就是壁垒
- **用户需求明确**：设计师和视频创作者是最愿意付费学习 AI 的群体
- **串联价值高**：单个工具教程很多网站都有，但"从概念→出图→视频→多平台适配"的完整流水线只有系统专题能做

## 2. 专题 A 知识结构：游戏运营 AI 方案全书

### 2.1 专题定位

- **一句话定位**：从活动策划到数据复盘，游戏运营人的 AI 全流程方案手册
- **目标用户**：产品运营(product-ops)、产品策划(product-planner)、数值策划(numerical-planner)
- **核心价值**：不教工具操作，教任务完成——每篇内容对应一个真实运营任务，给出完整 AI 解决方案

### 2.2 知识架构（4 层 × 每层 3 模块）

```
第一层：日常运营提效
├── 模块 1.1：AI 辅助活动策划（方案生成→奖励设计→排期优化）
├── 模块 1.2：AI 辅助需求文档撰写（PRD 框架→用户故事→验收标准）
└── 模块 1.3：AI 辅助日报/周报/复盘（数据整理→洞察提炼→汇报生成）

第二层：数据驱动决策
├── 模块 2.1：AI 数据分析与报表（Excel→洞察→可视化→自动报告）
├── 模块 2.2：AI 用户分析与分层（画像构建→行为分析→精准推送）
└── 模块 2.3：AI A/B 测试与效果评估（方案设计→数据解读→迭代建议）

第三层：增长与变现
├── 模块 3.1：AI 辅助用户留存策略（流失预警→召回方案→留存优化）
├── 模块 3.2：AI 辅助付费优化（价格测试→礼包设计→转化漏斗）
└── 模块 3.3：AI 辅助竞品分析与市场洞察（竞品拆解→趋势预判→差异化策略）

第四层：自动化与规模化
├── 模块 4.1：运营任务自动化工作流（n8n/Zapier→定时任务→批量执行）
├── 模块 4.2：AI Agent 辅助运营（客服自动化→数据监控→异常预警）
└── 模块 4.3：多模型协同运营方案（模型选型→任务分配→质量把控）
```

## 3. 专题 B 知识结构：AI 视觉内容生产指南

### 3.1 专题定位

- **一句话定位**：从创意到成品，游戏视觉工作者的 AI 生产流水线手册
- **目标用户**：平面设计师(graphic-designer)、视频设计师(video-designer)、新媒体运营(new-media)、品牌经理(brand-manager)
- **核心价值**：不只教单个工具，教完整的视觉生产流水线——每个环节用什么工具、怎么衔接、如何批量化

### 3.2 知识架构（4 层 × 每层 3 模块）

```
第一层：AI 出图基础
├── 模块 1.1：AI 图片生成入门（Midjourney / Nano Banana / Imagen 4 对比与选型）
├── 模块 1.2：Prompt 写图技巧（构图→风格→光影→文字渲染）
└── 模块 1.3：AI 图片编辑与修复（Photoshop AI / Canva AI / 局部重绘）

第二层：设计场景实战
├── 模块 2.1：游戏海报与 KV 设计（概念图→构图→排版→多尺寸适配）
├── 模块 2.2：社媒配图与广告素材（平台适配→批量变体→A/B素材测试）
└── 模块 2.3：角色/场景/UI 概念设计（风格探索→迭代→一致性控制）

第三层：AI 视频生产
├── 模块 3.1：AI 视频生成工具对比（Sora 2 / Kling 3.0 / Runway Gen-4.5 / Veo 3）
├── 模块 3.2：短视频脚本与分镜（选题→脚本→分镜→口播稿）
└── 模块 3.3：视频后期与特效（剪映 AI / Runway 特效 / Pika 特效 / 配音配乐）

第四层：视觉生产流水线
├── 模块 4.1：品牌视觉一致性管理（风格指南→AI约束→批量输出→审核机制）
├── 模块 4.2：多平台内容适配（抖音/B站/小红书/X 尺寸和风格差异）
└── 模块 4.3：AI 视觉内容工厂（批量生产→自动化流程→质量控制→版本管理）
```

## 6. 知识壁垒形成逻辑

### 6.1 三层壁垒模型

```
壁垒层 3（最难复制）
┌─────────────────────────────────────────────┐
│  系统化知识架构壁垒                          │
│  竞品可以抄单篇内容，但无法复制：             │
│  · 4层×12模块的知识体系设计                  │
│  · 教程×Prompt×工具的三维交叉索引             │
│  · 层级间的递进学习逻辑                      │
│  · 持续迭代形成的内容深度                    │
└─────────────────────────────────────────────┘

壁垒层 2（较难复制）
┌─────────────────────────────────────────────┐
│  行业垂直深度壁垒                            │
│  通用 AI 教程网站无法做到：                   │
│  · 每个方案都基于游戏行业真实场景              │
│  · Prompt 变量预填了游戏行业术语和框架         │
│  · 工具推荐基于游戏行业实测而非通用评测        │
│  · 案例全部来自游戏运营/设计真实工作           │
└─────────────────────────────────────────────┘

壁垒层 1（基础壁垒）
┌─────────────────────────────────────────────┐
│  内容密度与规模壁垒                          │
│  单纯的内容量已经形成门槛：                   │
│  · 专题A：25篇教程 + 180条Prompt + 10工具     │
│  · 专题B：24篇教程 + 120条Prompt + 18工具     │
│  · 总计约 50 篇深度内容 + 300 条实战Prompt     │
│  · 新进入者至少需要 3-6 个月追赶              │
└─────────────────────────────────────────────┘
```

### 6.2 壁垒持续加深机制

| 机制 | 做法 | 效果 |
|------|------|------|
| **时效更新** | 工具版本更新时同步更新对应模块内容 | 竞品抄了也很快过时 |
| **深度串联** | 每篇新教程自动关联到专题模块中 | 内容越多→专题越厚→壁垒越深 |
| **用户数据** | 通过 localStorage 记录学习进度 | 用户沉没成本→不愿迁移 |
| **独家组合** | 教程+Prompt+工具的三维组合是原创的 | 抄单维度容易，抄三维组合难 |
| **行业语境** | 所有示例用游戏行业真实数据和场景 | 通用 AI 站无法快速补齐行业深度 |

### 6.3 壁垒量化评估

| 指标 | 当前值 | 3 个月后目标 | 6 个月后目标 |
|------|:---:|:---:|:---:|
| 专题总内容量 | 50 篇 + 300 Prompt | 70 篇 + 400 Prompt | 100 篇 + 500 Prompt |
| 专题独有内容占比 | 30%（8 篇新建） | 50% | 60%+ |
| 知识模块完成度 | 60%（首批 10/模块） | 80% | 95% |
| 内容更新频率 | — | 每周 2 篇 | 每周 2-3 篇 |

## 附录：与现有体系的关系

### 专题 vs 专栏 vs 学习路径

| 维度 | 专栏(columns) | 学习路径(paths) | 专题(topics) |
|------|:---:|:---:|:---:|
| 组织逻辑 | 按工具/产品 | 按成长阶段 | 按工作场景 |
| 典型标题 | "Claude 深度使用指南" | "7天 AI 入门之旅" | "游戏运营 AI 方案全书" |
| 内容来源 | 专栏专属教程 | 跨专栏串联 | 跨全站串联 |
| 核心价值 | 掌握一个工具 | 建立基础能力 | 解决一类工作问题 |
| 内容深度 | 深（单工具纵深） | 浅→中（入门导向） | 中→深（任务导向） |
| 壁垒贡献 | 中（工具教程易被替代） | 低（路径设计易被模仿） | **高（系统化+行业化）** |

### 互相引流逻辑

```
学习路径（入口）──→ 完成基础后 ──→ 进入专题（深度学习）
                                    ↕
                              专栏（工具纵深）
                                    ↓
                           工具页/模型页（决策参考）
                                    ↓
                          Prompt 库（直接取用）
```

专题是全站内容的"高密度聚合层"，把散落的教程、Prompt、工具串联成有结构的知识产品。

第二部分：各模块内容职责定义
==============================

■ AI快报 (news)
  存在价值：让用户养成「每天来看一眼」的习惯
  不应做成：不要做成新闻聚合器，不堆无用信息
  对用户的帮助：只推与游戏岗位有关的AI动态，每条必须回答「这对我的工作意味着什么」
  自动化建议：采集→AI清洗→人工终审→发布（80%自动化）

  质量红线：
  - 每条必须有明确的 relatedJobs（不能全是 'all'）
  - summary 必须回答「so what」—— 这条新闻对目标岗位意味着什么
  - 72小时以上的旧闻不发（除非是重大里程碑）

■ 工具库 (tools)
  存在价值：帮用户快速找到「我的岗位该用什么工具」
  不应做成：不要做成大而全的工具目录，不收录与游戏行业无关的工具
  对用户的帮助：每个工具都能回答「它帮我解决什么问题」「上手有多难」「多少钱」
  自动化建议：监控新工具→AI初筛→人工评测→上架（50%自动化）

  质量红线：
  - 每个工具必须有真实的 problemsSolved（不能泛泛）
  - 必须有 difficulty 评级（基于真实试用）
  - pricing 必须准确且定期更新
  - 至少关联一个岗位场景

■ 模型库 (models)
  存在价值：帮用户理解「该选哪个模型」
  不应做成：不要做成技术参数罗列，不面向开发者
  对用户的帮助：普通人能看懂的模型对比——「我写文案用哪个」「我做图用哪个」
  自动化建议：监控模型更新→AI整理参数→人工验证→发布（60%自动化）

  质量红线：
  - bestFor 必须是具体场景，不是技术术语
  - desc 必须是一句话让非技术人员看懂
  - 参数变化时必须及时更新

■ 实战指南 / 教程 (tutorials)
  存在价值：这是整个平台最核心的内容资产
  不应做成：不要做成操作手册、不要做成视频搬运
  对用户的帮助：看完能直接解决一个真实工作问题
  自动化建议：选题AI辅助→人工撰写→AI校验→发布（20%自动化）

  质量标准（每篇必须满足）：
  1. 有明确的目标岗位
  2. 解决一个具体工作问题
  3. 有可操作的步骤（steps字段）
  4. 有预期产出（用户跟完能得到什么）
  5. 标注推荐使用的模型/工具

■ Prompt库 (prompts)
  存在价值：降低AI使用门槛，让用户「复制即用」
  不应做成：不要做成Prompt技巧理论文章
  对用户的帮助：直接复制到ChatGPT/Claude就能用，马上产出结果
  自动化建议：岗位场景→AI生成初版→人工实测优化→发布（40%自动化）

  质量红线：
  - 每个Prompt必须经过至少2个模型实测
  - 必须有example（示例输入）
  - 必须标注 recommendedModel（推荐模型）
  - {变量}标记清晰，用户知道替换什么

■ 资源库 (resources)
  存在价值：提供可下载的生产力工具包
  不应做成：不要做成网盘链接集合
  对用户的帮助：下载即可用的模板、指南、速查卡
  自动化建议：人工制作→定期更新→版本管理（10%自动化）

■ 岗位路径 (jobs)
  存在价值：这是全站的内容组织中枢——一切内容围绕岗位展开
  不应做成：不要做成岗位介绍页，不要做成JD
  对用户的帮助：「我是XX岗位，AI能帮我做什么」一目了然
  自动化建议：AI辅助关联内容→人工策划路径→定期优化（30%自动化）

==============================
第三部分：内容质量标准
==============================

■ 发布前检查清单（所有内容必须通过）

  □ 岗位相关性：这条内容是否明确服务至少一个岗位？
  □ 问题导向：这条内容是否回答了一个真实的工作问题？
  □ 可操作性：用户看完后是否知道下一步该做什么？
  □ 时效性：信息是否是最新的？（特别是pricing、版本号）
  □ 差异化：这条内容是否提供了网上不容易找到的视角？

■ 内容深度判断标准

  浅（不发）：「ChatGPT是一个AI工具，可以帮助你写文案」
  中（可发）：「投放经理可以用ChatGPT批量生成广告文案，方法是…」
  深（优先发）：「投放经理用ChatGPT将广告文案CTR提升23%的完整方法论，含Prompt模板和AB测试流程」

■ 不发布的内容类型

  × 无岗位关联的泛AI新闻
  × 纯技术参数罗列（没有「so what」）
  × 过时超过30天的工具/模型信息
  × 未经实测的Prompt模板
  × 与游戏行业无关的内容

第四部分：自动化内容流转规则
==============================

■ AI快报流转

  采集（RSS/API监控AI资讯源，每小时）
  → 清洗（AI去重、过滤无关内容）
  → 分类（AI自动分配 category）
  → 打标签（AI自动匹配 relatedJobs + tags）
  → 增强（AI生成 summary，回答「对岗位意味着什么」）
  → 人工终审（5分钟/条，确认发布）
  → 发布
  → 30天后归档（isHot自动降级）

■ 工具库流转

  监控（Product Hunt / 行业媒体 / 竞品动态）
  → AI初筛（是否与游戏行业相关）
  → 人工试用评测（difficulty、problemsSolved 基于实测）
  → 结构化录入（填充完整字段）
  → 关联岗位和教程
  → 发布
  → 季度Review（pricing更新、淘汰过时工具）

■ 模型库流转

  监控（厂商公告、开源社区、评测榜单）
  → AI整理参数变化
  → 人工验证+撰写bestFor
  → 更新或新增条目
  → 发布
  → 版本迭代时即时更新

■ 教程选题流转

  选题池（岗位痛点 + 工具更新 + 用户反馈 + 搜索数据）
  → AI辅助生成大纲
  → 人工撰写完整内容
  → 实操验证（按步骤走一遍）
  → AI校验（检查完整性、标签准确性）
  → 发布
  → 持续迭代（工具更新时同步更新教程）

■ Prompt生成流转

  场景识别（从岗位scenarios中提取）
  → AI生成初版Prompt
  → 多模型实测（ChatGPT + Claude 各跑一遍）
  → 优化迭代（根据输出质量调整）
  → 补充example和notes
  → 标注recommendedModel
  → 发布
  → 用户反馈驱动迭代

■ 资源沉淀流转

  内容积累到阈值 → 整理为可下载文档
  教程10篇+ → 输出为岗位指南PDF
  Prompt20个+ → 输出为Prompt手册
  工具评测积累 → 输出为工具速查卡
  → 定期更新版本号

日常运营指令
==============================

当用户说「重点：XXX」时的执行流程：

1. 解析重点方向（哪个模块 / 哪个岗位 / 哪种内容类型）
2. 评估当前该方向的内容状态
3. 按照上述质量标准判断内容缺口
4. 直接修改 data.js 中的内容
5. 确保修改后的内容通过发布前检查清单
6. 报告修改结果

示例：
- 「重点：投放经理」→ 增强投放经理的scenarios、补充相关Prompt、新增快报
- 「重点：Prompt质量」→ 为所有Prompt补充recommendedModel、优化Prompt文本
- 「重点：快报更新」→ 新增3-5条有深度的AI快报
- 「重点：视频设计」→ 增强视频设计师相关的工具、教程、Prompt
- 「重点：首页体验」→ 优化jobRecommendations和editorsPicks数据
